## Task Workflow — Spec → Plan → Code

For any non-trivial task, follow this sequence strictly:

1. **Agree on the spec** — clarify requirements, constraints, edge cases with the user before planning
2. **Write a visible implementation plan** — markdown file or structured outline the user can read and edit; save it to the repo so the user owns it
3. **Get explicit approval** — wait for the user to confirm the plan before writing any code
4. **Implement step by step** — one logical chunk at a time, not a big-bang generation

For small tasks where a full plan would be overkill: state the approach in one sentence and confirm before coding.

Never jump straight to code.
