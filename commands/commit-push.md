Выполни следующие шаги:

1. Покажи `git status` — пойми что изменилось и что не отслеживается.
2. Проверь что среди изменённых файлов нет секретов (.env, credentials, private keys, *.pem). Если есть — останови и предупреди.
3. Запусти тесты если они есть:
   - Python: `pytest`
   - Go: `go test ./...`
   - Node: `npm test -- --watchAll=false`
   Если тестов нет — пропусти. Если тесты падают — останови и покажи ошибки.
4. Создай новую ветку от текущей с осмысленным именем (формат: `feature/краткое-описание` или `fix/краткое-описание`).
5. Добавь файлы в staging **явно по именам** (не `git add .` и не `git add -A`).
6. Сделай коммит с осмысленным сообщением на английском (conventional commits: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`).
7. Запуши ветку.
8. Создай Pull Request через `gh pr create` с описанием:
   - **Summary** — что изменилось (буллеты)
   - **Motivation** — зачем
   - **Changed files** — таблица файлов с кратким описанием

Используй gh cli для создания PR. Если gh не установлен — скажи мне.
